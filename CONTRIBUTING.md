Thank you for your interest in contributing to the centralized repository.  
This document outlines the guidelines for contributing to the **Shipping Rates and Calculations** project.

---

## Code Style

Please follow the coding style and conventions used in the existing codebase.  
Maintaining a consistent style helps improve readability and long-term maintainability.

---

## Documentation

Ensure that all contributions are well-documented.

- Add clear and meaningful comments where necessary
- Provide a concise and descriptive explanation of your changes in the pull request

---

## Testing

Before submitting a pull request:

- Thoroughly test your changes
- Include relevant test cases when applicable
- Ensure that all existing tests pass successfully

---

## Issue Tracker

Please review the issue tracker for open issues or feature requests.

- If you plan to work on an issue, comment on it to inform others
- This helps avoid duplicated efforts and improves collaboration

---

## Code Review

All contributions are subject to a code review process.

- Be open to feedback and suggestions
- Be willing to revise your code if required  

Code reviews help maintain high code quality and project consistency.
